# 🚪 API Gateway (Halfi Core)

Этот модуль является **единой точкой входа** для всех микросервисов проекта Halfi (auth-service, notification-service и т.д.).
Построен на базе **Spring Cloud Gateway**. Вместо того чтобы клиент (мобильное приложение) обращался к каждому микросервису по разным IP и портам, он отправляет **все** запросы сюда (на порт `8081`), а Gateway сам "под капотом" маршрутизирует их в нужный контейнер.

---

## 📡 Основные Эндпоинты (REST API)
Все запросы маршрутизируются через API Gateway. Базовый URL для мобильного приложения или фронтенда:
👉 `http://localhost:8081/api/auth`

| Метод | Эндпоинт | Описание | Тело запроса (JSON) / Параметры |
|---|---|---|---|
| `POST` | `/register` | Регистрация нового аккаунта | `{"email": "my@email.com", "password": "123"}` |
| `POST` | `/login`| Вход в систему и получение JWT | `{"email": "my@email.com", "password": "123"}` |
| `POST` | `/verify` | Активация кода с почты | **Params:** `?email=my@email.com&code=1234`<br/>*(Важно: если передаете через URL строку вручную, символ `+` нужно кодировать как `%2B`)* |
| `POST` | `/forgot-password` | Запрос на сброс забытого пароля | `{"email": "my@email.com"}` |
| `POST` | `/reset-password` | Установка нового пароля по коду | `{"email": "my@email.com", "code": "1234", "newPassword": "new"}` |
| `POST` | `/resend-code`| Запросить повторное письмо на почту | `{"email": "my@email.com"}` |

## ⚙️ Конфигурация и Маршруты

- **Порт**: `8081` (основной и единственный порт, к которому коннектится Frontend/Mobile)
- **Стек**: Kotlin 1.9.22, Spring Boot 3.3.5, Spring Cloud Gateway

**Таблица маршрутизации (Routes):**
| Вызов от клиента | Куда перенаправляется редирект | Описание маршрута |
|--------------|----------------|----------|
| `/api/auth/**` | `auth-service:8084` | Регистрация, логин (JWT), переотправка, сброс пароля |
| `/api/user/**` | *(В планах)* | Профиль, изменение аватарки и данных пользователя |

*Примечание: Gateway и остальные сервисы общаются между собой по внутренней Docker-сети, поэтому открывать порты микросервисов наружу не обязательно.*

## 🚀 Запуск через Docker

Так как проект перешел на полностью контейнеризированную архитектуру, API Gateway больше не стоит запускать отдельно через IDE. Он должен работать внутри единой сети `halfi_network` вместе с Kafka и БД.

### Самый быстрый способ полного запуска
Запустить весь кластер всех проектов одной командой (из корневой папки Halfi_core):
```bash
bash docker/setup.sh
```

### Ручной запуск (Только API-Gateway)
Если вы поправили логику только в Gateway и хотите перезапустить лишь его:

1. **Создание общей сети** (если вы сбросили докер и сети нет):
   ```powershell
   docker network create halfi_network
   ```
2. **Сборка актуального JAR-файла**:
   ```powershell
   .\gradlew.bat :api-gateway:clean :api-gateway:bootJar -x test
   ```
3. **Запуск контейнера Gateway**:
   ```powershell
   docker compose -f docker/apps/docker-compose.apigateway.yml up --build -d
   ```

### 🛑 Как остановить
```powershell
docker compose -f docker/apps/docker-compose.apigateway.yml down
```

## 🩺 Проверка статуса (Health Check)
Чтобы убедиться, что Gateway поднялся и прогрузился:
Перейдите по ссылке (в браузере или Postman):
👉 `http://localhost:8081/actuator/health`

**Ожидаемый ответ:** `{"status":"UP"}`
